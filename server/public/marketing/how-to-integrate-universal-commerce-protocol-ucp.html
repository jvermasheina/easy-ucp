What is UCP and How Does It Work?

Universal Commerce Protocol (UCP) is an open-source protocol announced by Google for agentic commerce - enabling AI agents to complete purchases on behalf of users.

UCP is NOT a product feed. Unlike traditional shopping feeds that list products, UCP handles the checkout process: cart management, tax calculation, payment processing, and order fulfillment.

When a user asks Gemini to "buy wireless headphones under $100," the AI agent: 1. Discovers your store's UCP profile at `/.well-known/ucp` 2. Creates a checkout session via your UCP endpoints 3. Handles payment securely using credential exchange 4. Receives order confirmation and tracking updates

UCP was co-developed with Shopify, Etsy, Wayfair, Target, and Walmart, and is endorsed by 20+ companies including Stripe, Visa, Mastercard, and PayPal.

Key Takeaways:

  • UCP handles checkout and orders, not product discovery
  • AI agents discover your store via `/.well-known/ucp` endpoint
  • Supported by major retailers and payment providers
  • Open-source protocol available on GitHub

Technical Requirements

Before implementing UCP, ensure your platform meets these requirements from the official specification:

Infrastructure Requirements

HTTPS Only: All UCP communication must use HTTPS

Discovery Endpoint: Publish profile at `/.well-known/ucp`

API Capabilities: REST API or Model Context Protocol (MCP) binding

Webhook Support: Receive order lifecycle events (shipped, delivered, canceled)

Data Format Requirements

Currency: Amounts in minor units (cents) - e.g., $10.00 = 1000

Dates: RFC 3339 format - e.g., `2026-01-15T10:30:00Z`

Authentication: OAuth 2.0 for user authorization

Signatures: JWK public keys for webhook verification

Setting Up Your UCP Profile

Your UCP profile is published at `/.well-known/ucp` and tells AI agents what your store supports.

Step 1: Create Profile JSON

Create a JSON file at `https://yourstore.com/.well-known/ucp` with your capabilities:

{
  "version": "2026-01-11",
  "name": "YourStore",
  "capabilities": [
    {
      "name": "dev.ucp.shopping.checkout",
      "version": "2026-01-11",
      "spec": "https://ucp.dev/services/shopping/checkout",
      "schema": "https://ucp.dev/services/shopping/checkout.schema.json",
      "rest": {
        "endpoint": "https://yourstore.com/api/ucp",
        "schema": "https://ucp.dev/services/shopping/rest.openapi.json"
      }
    }
  ],
  "webhook": {
    "url": "https://yourstore.com/webhooks/ucp",
    "publicKey": {
      "jwk": { /* your JWK public key */ }
    }
  }
}

Step 2: Implement Discovery

Serve your profile with proper headers. Example Express.js route:

app.get('/.well-known/ucp', (req, res) => {
  res.setHeader('Content-Type', 'application/json');
  res.setHeader('Access-Control-Allow-Origin', '*');
  res.json({
    version: '2026-01-11',
    name: 'YourStore',
    capabilities: [/* ... */]
  });
});

Implementing Checkout Capability

The checkout capability handles cart creation, tax calculation, and purchase completion. Based on the UCP specification, implement these endpoints:

POST /checkout/session - Create Session

AI agent calls this to start a checkout:

// Request
{
  "items": [
    {
      "productId": "headphones-123",
      "quantity": 1,
      "priceMinorUnits": 7999
    }
  ],
  "shippingAddress": {
    "country": "US",
    "postalCode": "94105"
  }
}

// Response
{
  "sessionId": "sess_abc123",
  "subtotal": 7999,
  "tax": 720,
  "shipping": 500,
  "total": 9219,
  "currency": "USD"
}

POST /checkout/complete - Complete Purchase

Called when user authorizes payment:

// Request
{
  "sessionId": "sess_abc123",
  "paymentToken": "tok_visa_xxx",
  "paymentProvider": "stripe"
}

// Response
{
  "orderId": "order_789",
  "status": "confirmed",
  "confirmationNumber": "ABC-123-XYZ",
  "estimatedDelivery": "2026-01-20T00:00:00Z"
}

Authentication and Security

UCP uses OAuth 2.0 for user authorization and JWK signatures for webhooks, per the security guidelines.

OAuth 2.0 Identity Linking

Allow users to link their store account with AI agents:

1. Platform sends authorization request 2. User logs in and grants permissions 3. Your store returns OAuth token 4. Agent uses token for future purchases

This enables agents to access user's saved addresses, payment methods, and order history.

Webhook Signatures

Verify webhook requests using JWK public keys:

1. Platform signs webhooks with private key 2. Include signature in `UCP-Signature` header 3. Verify using platform's public JWK from their UCP profile 4. Reject unsigned or invalid webhooks

This prevents spoofed order updates and ensures authenticity.

3. Implement checkout endpoints using WC_Cart and WC_Order

5. Test endpoints with UCP validator

Magento

No official support

Workaround: Custom module development using Magento APIs

1. Create Magento 2 module
2. Add route for `/.well-known/ucp`
3. Use Quote and Order APIs for checkout
4. Implement OAuth via Magento's built-in OAuth 2.0
5. Deploy webhooks using Magento event observers

Workaround: Direct implementation following official spec

1. Review UCP specification
2. Use SDKs from GitHub (Node.js, Python, Go)
3. Implement required capabilities (checkout, identity, order)
4. Test with reference implementations
